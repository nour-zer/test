PROJECT IDEA

Overview

The proposed project is a cloud based platform initially titled “Green My Home” or GMH for short. GMH’s aim is to promote sustainability in all Australian homes. It will be designed and implemented in a phased approach. Phase 1 will focus on solar panels and home energy consumption and patterns, whilst subsequent phases will evolve the features by incorporating sustainable items such as heat pumps, air tightness, shading, orientation etc. For the purposes of this project will focus our attention on phase 1 only which will form our baseline.

GMH-Phase 1 will have IoT power meters that will measure and graph a household’s energy consumption, which will feed into the GHM platform. The GHM platform will then perform some analysis and provide the user with recommended options for solar panels and inverter systems that are tailor made for the particular household.

Motivation

According to Australian Renewable Energy Agency (ARENA) only 21% of Australian homes have some sort of solar system installed. The aforementioned statistic clearly demonstrates that there is a long way for Australians to be sustainable. From a financial standpoint, Solar systems present Australian families an opportunity to save significantly on their electricity bills and in some instances, they can actually make some money by providing excess electricity supplied from their own solar system back to the grid.

One of the biggest challenges for Australian families in taking up a solar system is that there is a multitude of options and combinations of solar panels and inverters, compounded with varying customer objectives and site specific factors, this makes choosing the perfect system near impossible. Enter GMH, this platform removes the “guess work” by collecting real life statistics, deep analysis combined with other parameters to provide the household with the ultimate customised solution.

In summary, there is no one size fits all when it comes to solar energy systems and GMH is here to address this very problem.

Description

GHM Phase 1 will be composed of the following features:

  • Household electricity usage & reporting
  • Household user/customer portal (Frontend)
  • Data analysis software engine
  • Databases (Backend)
  • Solar system supplier & installer portal (Frontend)

Household electricity usage & reporting

    In order to obtain data about the electricity usage of a particular household we need to somehow collect this data, this can potentially be done using 2 methods:
  • Leverage of existing smart power meter and creating an interface with the electricity authority.
  • Install an independent IoT power meter that has already been preconfigured to interface with the GHM platform.
  • Once data collected, it will also provide reports and patterns of the household’s energy consumption.

Household user portal

    GHM will provide end users with a portal where they are able to do the following:
  • Create a user account.
  • Fill out a questionnaire that will inform GHM what are the parameters and objectives of the household. For example, what is the budget? How big is the roof space? What is your desired return on investment?
  • Access historical & real time electricity usage
  • View different solar system options.
  • Request a quote and interact with vendors
  • Write supplier and product reviews
  • Interact with other users/community who are interested in solar and sustainability in general.

Data analysis software engine

The crux of the GHM platform is its ability to analyse all data and provide the user with a recommended end to end solar system. It considers various input data such as total daily energy consumption, available roof space, budget and myriad of other factors and data.

Databases

In order to store, search & retrieve data we require backend database servers. The databases will store information such as customer details, supplier details, product details and energy usage for each customer and so forth.

Solar system supplier portal

    Similar to the user or customer portal, vendor or supplier portal is needed. This portal will facilitate the following for each vendor:
  • Create and update supplier account
  • Update their company details
  • Upload and update various products and services including specifications
  • Interact with customers and provide them with quotes

To tie it all together, GMH is a one stop shop that connects customers with suppliers of solar system products. Architecturally speaking, It has front end web based user interface for both the customers and supplier, on the back end it has databases and data analytics which are used to store various pieces of information which thus in turn is analysed and corresponding result or recommendation is presented to the end user. It also has a blog, chat and forum facility in which different users are able to interact with one and other and talk all things green. As mentioned earlier, this is only the first phase of the platform, as the product becomes more mature and fruitful then we can consider evolving it to the next level by incorporating other sustainability centric elements such as heat pumps, HRV and so forth.

Tools and Technologies

In order for us to implement the GMH platform we require to have a cloud provider such as Amazon Web Services or AWS for short hosting the platform. Within this cloud environment we require virtual servers, storage and databases to build the environment. For data collection of the power consumption there are a few options but one option we can look at exploring is a product called Powerpal Energy monitor. This product does not require an electrician, you simply plug it into your power meter via an optical cable and in real time it sends energy used.

Skills Required

To deliver the project end to end there are a few skills and expertise required. We require a cloud engineer to set up the cloud environment which the platform will be hosted in. We also require a software engineer who is able to design the entire system and draw up a system diagram showing all interfaces/API etc. Similarly we need the UX designer to design the layout of the portals. Additional skills that are needed are business analyst where they are able to gather the requirements and present them as user acceptance criteria. Lastly but definitely not least, a software developer is needed to code and develop the GMH platform. As a whole, finding all these skills is possible as they are all ubiquitous roles commonly found in the IT industry.

Outcome

If the project was successful, then there will be a huge uptake of Australian homes having solar installed in their homes which will lead to an amazing outcome, with Australia leading the way in becoming fossil fuel free that will have tremendous positive impacts on the environment and future generations. It will also help save Australians lots of money on electricity running costs. If GMH has great success domestically then I see no real blockers for it to be rolled out globally.

In short, when it comes to sustainability and solar systems there is no one size fits all, as each situation and requirements are different. GMH makes choosing the right system easy and straightforward which will in turn hopefully promote more sustainable homes, which will make the world a better place now and in the future.
